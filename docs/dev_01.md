# 작업 계획서: Regex 테스트 툴 구현 (dev_01)

## 1. 개요
개발자들을 위한 All-in-One 유틸리티 웹사이트에 정규표현식(Regex)을 테스트하고 분석할 수 있는 툴 페이지를 추가합니다.

## 2. 요구 사항 및 UI 구성
**[메뉴]** 
- 사이드바 네비게이션에 `Regex` 메뉴 추가

**[메인 컨텐츠 구조]**
- **상단 영역 (Input Area)**
  - 정규표현식 문법을 입력하는 Input/Textarea.
  - 우측에 `Run` (실행) 버튼 배치.
- **하단 영역 (Split View)**
  - **좌측 (Regex Analyzer):** 입력된 정규표현식을 분석하여 의미를 트리 구조(Tree Structure) 형태로 시각화 및 설명.
  - **우측 (Sample Data & Highlight):** 테스트할 샘플 텍스트를 입력하는 공간. `Run` 버튼 클릭 시 매칭되는 문자열의 영역이 시각적으로 하이라이팅 됨.

## 3. 기술적 구현 방안

### 3.1 라우팅 및 레이아웃 반영
- 현재 단일 페이지(App.tsx)로 구성된 레이아웃에 `react-router-dom`을 도입하여 메뉴별 페이지 이동이 가능하도록 구조 개선.
- `/regex` 경로에 Regex 테스트 툴 컴포넌트 마운트.

### 3.2 Regex 분석기 (Tree View)
- 정규표현식을 파싱하여 AST(Abstract Syntax Tree) 형태로 변환해주는 라이브러리로 `regexp-tree`를 채택하여 도입.
- 파싱된 트리를 재귀적으로 순회하며, 각 토큰(문자, 수량자, 그룹 등)에 대한 설명을 렌더링.

### 3.3 샘플 데이터 하이라이팅
- 샘플 텍스트 영역은 사용자가 직접 타이핑할 수 있어야 하므로 `<textarea>` 혹은 contenteditable 렌더러를 사용.
- **하이라이팅 기법:** 사용자가 입력한 정규표현식을 `new RegExp()`로 생성하여 샘플 데이터에 `match()` 또는 `exec()` 수행.
- 매칭된 인덱스 정보를 바탕으로 원본 텍스트를 여러 개의 `<span class="bg-yellow-200">`과 같은 조각으로 나누어 화면에 오버레이 혹은 렌더링.

### 3.4 상태 관리
- `regexPattern`: 사용자가 상단에 입력한 정규식 문자열
- `sampleText`: 사용자가 우측에 입력한 테스트 대상 문자열
- `isRegexValid`: 정규식 문법 오류 여부 (오류 발생 시 UI 피드백 제공)
- `runCount`: Run 버튼 클릭을 감지하여 하단 분석/매칭 결과 업데이트 트리거

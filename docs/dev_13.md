# dev_13: 메인 대시보드(홈) 페이지 신설 및 타이틀 네비게이션

## 개발 목표
- `tools.oofbird.me` 서비스에 최초 접속(루트 경로 `/`)할 때 제공되는 고급스러운 대시보드 홈 화면을 신설합니다.
- 사이드바 메뉴가 불필요하게 복잡해지지 않도록 메인 페이지를 별도로 추가하지 않고, 헤더 영역의 로고 타이틀을 클릭할 때 홈 화면으로 이동할 수 있도록 설계합니다.
- 전체적인 UI 톤앤매너에 맞춘 프리미엄 카드 디자인과 호버 애니메이션을 적용합니다.

## 주요 변경 사항

### 1. 메인 대시보드 페이지 구현 (`src/pages/Home.tsx` 신설)
- **카드형 도구 소개 그리드**: 제공 중인 모든 개발자 도구(5종)를 카드 레이아웃 형태로 배치했습니다.
  - 정규표현식 테스트 도구 (Regex Tester)
  - 시간/날짜 변환기 (Epoch Converter)
  - 포맷터 및 변환기 (JSON/XML Formatter)
  - 텍스트 인코더 & 해시 (Text Encoder & Hash)
  - 이미지 Base64 변환기 (Image & Base64)
- **프리미엄 UI/UX 요소**:
  - 각 도구의 직관적인 디자인 아이콘과 맞춤형 그라데이션 그라데이션 아바타(`Avatar`) 배경을 적용했습니다.
  - 마우스 호버 시 위로 살짝 떠오르며 그림자가 짙어지는 리프팅 효과(`translateY(-6px)` 및 `boxShadow`) 및 테두리 색상 강조를 통해 시각적 인터랙션을 개선했습니다.

### 2. 라우팅 및 타이틀 클릭 네비게이션 적용 (`src/App.tsx` 수정)
- **루트 라우트 설정**: 기존에 `/` 접속 시 `/regex`로 리다이렉트하던 구문을 제거하고 신설된 `<Home />` 컴포넌트가 렌더링되도록 수정했습니다.
- **헤더 타이틀 링크**: AppBar 내의 `OOFBIRD Dev Tools` 타이틀 텍스트를 `react-router-dom`의 `<Link>` 컴포넌트로 래핑하여 클릭 시 홈 화면(`/`)으로 이동할 수 있도록 동선 구조를 수정했습니다.

### 3. 컴파일 오류 수정 및 MUI v6 호환성 확보
- `src/App.tsx` 내 미사용 중이던 `Navigate` 임포트 경고를 정리했습니다.
- MUI v6 환경 하에 구버전 Grid 문법(`item`, `container`, `justifyContent`)으로 인해 발생하던 빌드 오류를 MUI v6 표준인 **Grid2 (Grid)** 문법(`size`, `sx` 활용)으로 통일 및 리팩토링하여 프로덕션 빌드 컴파일을 정상화(`0 errors`)하였습니다.

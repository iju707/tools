# Regex Tool 기능 개선 계획서 (dev_02)

## 1. 개요
현재 Regex Tool은 입력된 정규표현식을 기반으로 텍스트의 일치(Match) 영역을 찾아 하이라이팅하는 기능을 제공합니다.
여기에 캡처 그룹(Capture Group) 패턴이 적용되었을 경우, 정규식 그룹에 매치된 세부 항목들을 분리하여 화면에 표시하는 기능을 추가합니다.

## 2. 요구 사항
- 정규표현식 내에 `()`를 사용한 캡처 그룹이 있는 경우, 매칭된 결과 텍스트 외에 그룹별 일치 텍스트도 추출해야 합니다.
- 우측 하단의 매칭 결과 화면(Test String & Matches)에 추출된 그룹 요소를 사용자가 쉽게 알아볼 수 있도록 표시해야 합니다.

## 3. 구현 상세
### 3.1. 매치 로직(Engine) 업데이트
- `RegexTool.tsx`의 `useMemo` 기반 매치 로직 수정
- `RegExp.exec()` 반환 배열의 인덱스 1번부터 마지막까지의 값을 순회하며 그룹 배열(Groups) 생성
- (선택 사항) `match.groups` 프로퍼티를 확인하여 Named Capture Group이 존재할 경우 이름과 값을 매핑하여 저장

### 3.2. UI/UX 업데이트
- 결과 화면 영역 하단에 "Match Details" 패널 추가
- 리스트 형태로 "Match 1", "Match 2" 구조를 만들고, 각 매치 하위에 "Group 1: 값", "Group 2: 값" 형태로 렌더링
- Tailwind CSS를 활용해 기존 디자인과 어우러지도록 깔끔한 카드(Card) 스타일 또는 리스트 아이템 적용

## 4. 기대 효과
- 복잡한 데이터 추출 시, 사용자가 작성한 정규식이 의도한 대로 데이터를 그룹화하여 가져오는지 직관적으로 검증할 수 있습니다.

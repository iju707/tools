# dev_16: 다중 크론탭 규격 지원 (Standard, Spring, AWS, Quartz)

## 개발 목표
- 기존의 표준 리눅스 5필드 크론 규격에서 더 나아가 백엔드 및 클라우드 인프라 설계(Spring Scheduler, AWS EventBridge, Quartz Scheduler 등)에서 실무적으로 활용되는 다양한 크론탭 규격(5필드, 6필드, 7필드)을 모두 자유롭게 조작하고 번역할 수 있는 **다중 크론탭 엔진**을 완성합니다.
- 규격 변경 시 UI 슬라이더 및 입력 필드, 예측기, 번역기 등의 핵심 엔진들이 모두 동적으로 전환되도록 설계하며, 전환 과정에서 발생하는 React의 무한 상태 갱신 오류(Minified React Error #185)를 해결해 신뢰할 수 있는 사용자 환경을 완성합니다.

---

## 주요 구현 사항

### 1. 4대 크론 규격 완벽 대응 (`src/pages/CronTool.tsx`)
- **다중 규격 셀렉터**: 상단 헤더 우측에 MUI Select Dropdown을 도입하여 네 가지 주요 스케줄러 시스템 규격을 실시간으로 변경 가능하게 했습니다.
  1. **표준 Linux/Unix (5필드)**: `분 시 일 월 요일`
  2. **Spring Scheduler (6필드)**: `초 분 시 일 월 요일`
  3. **AWS EventBridge (6필드)**: `분 시 일 월 요일 연도` (일/요일 중 하나는 반드시 `?` 지정 제약)
  4. **Quartz Scheduler (7필드)**: `초 분 시 일 월 요일 연도` (일/요일 중 하나는 반드시 `?` 지정 제약)
- **규격별 가변 프리셋 및 비주얼 제네레이터**:
  - 선택한 규격(Standard, Spring, AWS, Quartz)에 딱 부합하는 **맞춤형 프리셋 칩** 리스트가 실시간 전환됩니다.
  - 비주얼 탭 및 고급 커스텀 탭에서 각 규격에 맞는 **입력 필드 개수(5개 ~ 7개)와 설명 레이블(초, 연도 필드의 동적 활성화)**을 정교하게 다원화시켰습니다.

### 2. 크론 코어 엔진 고도화 (`validateCron`, `parseCronToKorean`, `getNextExecutions`)
- **유효성 검증(`validateCron`) 확장**:
  - 각 규격의 필드 개수 검증 규칙을 추가했습니다.
  - AWS 및 Quartz에서 강제되는 "일(DOM)과 요일(DOW) 중 하나는 반드시 `?`여야 하며, 둘 다 `?`일 수는 없음" 제약 요건을 완벽하게 검출합니다.
  - 초(0-59), 연도(1970-2099) 필드의 올바른 정규식 매칭을 정밀화했습니다.
- **번역기(`parseCronToKorean`) 정밀화**:
  - 초(Second) 및 연도(Year) 정보를 추출하여 매끄러운 한국어로 구성합니다 (예: `0/5 * * * * ? *` -> "매 5초마다").
  - AWS/Quartz의 특수 문자 `?`, `L` (해당 월의 마지막 날 또는 마지막 요일), `#` (n번째 요일)에 대응하는 고도화된 한글 직관 문장을 완성했습니다.
- **예측기(`getNextExecutions`) 연산 안전 확보**:
  - 초 단위 스케줄 분석 시 1초 단위로 미래 실행 시각을 빠르게 탐색하되, 루프 성능 안전장치 및 2099년 연도 상한선 체크 코드를 통해 연산 폭주를 예방했습니다.

### 3. 고급 커스텀 탭 렌더 크래시 해결 및 UI 공간 최적화
- **무한 렌더 루프 차단 (React Error #185)**:
  - 비주얼 컨트롤 탭에서 고급 커스텀 탭으로 전환하는 찰나에 양방향 `useEffect`가 어긋나 무한 상태 동기화 핑퐁을 돌던 문제를, 크론식 변경 시 항상 개별 `customFields` 상태를 일관성 있게 자동 동기화하도록 유도하여 원천 차단했습니다.
- **예측 목록 개수 조정 (5회 -> 3회)**:
  - 모바일 및 가로형 다양한 뷰포트에서 하단부에 불필요한 세로 스크롤이 생겨 디자인 일체감이 떨어지는 현상을 최적화하기 위해, 예측 리스트 표시 개수를 3개로 한층 밀도 있게 개선했습니다.
- **빌드 및 TypeScript 무오류 통과**:
  - `npm run build`를 완벽히 통과시켜 프론트 빌드 아티팩트 정합성을 입증했습니다.
